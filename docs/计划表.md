### 总览
该项目旨在做一个轻量数据存储引擎，即数据库。目前暂定只有四个基础功能，增加，删除，查找，改正
### 存储数据结构
使用 SkipList 作为内存数据结构，使用 LSM 架构完成整体存储结构和实现方式。

#### 阶段1：

5. 明确数据流向

#### 阶段2：
阶段二的目标是做出一个最小可行产品。能完成非常小范围数据的增删查改这样的任务，能够持久化存储。我将阶段2拆分成两部分，一部分是纯内存阶段，该阶段只需要考虑内存中的数据，不做持久化。第二部分是将内存中的数据持久化，得到最终的可运行产品。
##### A：内存闭环
1. 对外接口模块的完成
   - 对外接口的完成应该走在核心模块的前面，在这边我们需要做简单的增删查改，相应值等。所以我们提前完成这些功能。
   - 明确KV语句CRUD格式，明确Key-Value格式
2. 信息处理模块
   - 信息处理模块往往与对外接口模块联动，用于处理KV语句，用于处理Key-Value。
3. MemTabel的实现
   - 完成跳表的节点结构，随即层高，查找、插入以及遍历。
   - 完成Arena内存池，使之能正确给跳表节点分配空间。
   - 将跳表封装为Memtable模块，开放使用接口，而不是直接操作跳表。
4. 基础测试
   - 测试是否能有效开辟Arean内存池，能否正常分配。
   - 能否检测MemTable大小。
   - 能否有效查询，包括查询有效值，查询删除值，以及查询无效值。
   - 能否增加元素，删除元素，重复加入，重复删除，覆盖元素等。
   - 能否遍历跳表。
##### B：持久化闭环
该阶段目标为将内存中临时数据持久化到磁盘，不过目前阶段不考虑合并压缩，清理等问题。
1. 实现WAL，请求先走WAL，再走内存。这边设计已归档WAL与未归档WAL，配合后续刷盘，当刷盘完成，将WAL归档。做到断电后能自动恢复未刷盘数据。
2. 实现Immutable MemTable与Flush。当MemTable容量达到阈值时，将之冻结，并再后台进行Flush刷盘，并将WAL归档。当有新的写入时，使用新的Arean内存池分配新的skipList节点。
3. 实现SSTable v1。这里仅仅做一个区间边界，不做BloomFliter，也不处理SSTable，目标就是将之存入磁盘中，可以找到，可以读取。
4. 完善查询路径。将查询路径补足到能进行SSTable的查询。
5. 测试功能，能否正常读取SSTable中的内容。
