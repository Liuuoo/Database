### 总览
该项目旨在做一个轻量数据存储引擎，即数据库。目前暂定只有四个基础功能，增加，删除，查找，改正
### 存储数据结构
使用 SkipList 作为内存数据结构，使用 LSM 架构完成整体存储结构和实现方式。
### 落实
#### 阶段0:
1. 该阶段主要构建项目骨架，目前根据AI的建议，搭建了如下骨架：
   - docs/ --用来存放一些文件
   - src/  --主要代码
   - include/ --项目需要的一些库
   - tests/ --测试文件夹
   - CMakeLists.txt 
    
    至于src中，可能会包括各个模块，比如page，disk...
2. 完成 CMakeLists，使之能正常 build/run，可以成功编译一个空的程序。

3-30已完成阶段0，开始草拟、执行阶段1

#### 阶段1：
目标：完成存储引擎第一轮方案设计，明确系统范围、核心数据结构、数据流向与模块边界，为阶段2实现做准备。

1. 明确阶段1范围
   - 确定本项目当前阶段聚焦于轻量级单机存储引擎。
   - 明确阶段1以设计为主，不追求完整功能落地。
   - 明确阶段1暂不深入事务、并发控制、崩溃恢复、压缩合并等高级能力。

2. 明确核心数据结构方案
   - 确定使用 SkipList 作为内存数据结构。
   - 明确 SkipList 在内存写入、查询、有序遍历中的角色。
   - 明确其与整体 LSM 架构之间的关系。

3. 明确数据模型
   - 确定项目当前采用的基础数据模型。
   - 明确 key、value、记录状态等基本概念。
   - 明确插入、更新、删除三类写操作在逻辑上的区别与联系。

4. 明确 CRUD 语义
   - 规划 Create、Read、Update、Delete 的行为定义。
   - 明确查询结果以什么状态为准。
   - 明确删除是否采用删除标记这一思路。

5. 明确数据流向
   - 规划写入请求从进入系统到写入内存的大致路径。
   - 规划读取请求从内存到磁盘的查找顺序。
   - 规划删除与更新请求在系统中的处理路径。

6. 明确落盘策略
   - 规划内存数据何时需要落盘。
   - 规划落盘后磁盘文件在系统中的作用。
   - 明确阶段1只确定落盘策略，不展开复杂磁盘管理细节。

7. 明确模块划分
   - 规划对外接口层的职责。
   - 规划内存表模块的职责。
   - 规划磁盘读写模块的职责。
   - 规划记录结构与元信息的职责。

8. 形成阶段1产出
   - 一份数据模型说明。
   - 一份 CRUD 数据流转图。
   - 一份落盘策略说明。
   - 一份模块划分说明。
   - 一份阶段2实现顺序草案。

9. 阶段1完成标准
   - 能清楚说明系统当前要解决的问题和暂不解决的问题。
   - 能清楚说明 SkipList 在系统中的定位。
   - 能清楚说明 CRUD 的基本语义与数据流向。
   - 能清楚说明落盘的目的与触发时机。
   - 能清楚说明后续实现应先做哪些模块。
